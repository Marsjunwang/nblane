# 看板使用手册

看板是 nblane **私人操作系统（Private OS）** 里的 **当前执行面**：用本地文件 `profiles/<档案名>/kanban.md` 记录本周在推进什么、排队什么、已完成什么。可与 **技能树**、**证据池** 通过「已完成 → 证据」流程衔接。

**运行方式** 与侧栏选档案等通用说明见 [Web 使用手册 · §5](web-ui.md)。

---

## 1. 文件与真源

| 文件 | 作用 |
|------|------|
| `kanban.md` | 主看板（四列任务） |
| `kanban-archive.md` | 可选；从「已完成」**归档**出去的历史（见 [kanban-archive.md](kanban-archive.md)） |

在页面点 **保存** 或多数会自动落盘的操作后，内容会写回 `kanban.md`。**从文件重新加载** 会丢弃当前浏览器里未保存的改动，以磁盘为准。

---

## 2. 四列分别适合放什么

| 列（界面中文名） | 典型用途 |
|------------------|----------|
| **进行中** | 本周正在做的事；适合填 **背景**、**开始日期**、拆 **子任务** |
| **队列** | 接下来要做；可写 **原因**、**阻塞** |
| **已完成** | 已交付；可写 **结果**、**结束日期**；适合做 **AI 摄入** 的来源 |
| **也许 / 将来** | 低承诺的想法或远期项（无 checkbox 行，结构最简单） |

**完成状态** 由 **所在列** 与保存时的勾选位共同体现：用列上的 **移动按钮** 把任务挪到「已完成」即表示工作流上完成；不必再理解旧的「保持」类下拉菜单语义。

---

## 3. 新建任务（按列不同）

在某一列顶部的卡片里输入 **标题**，再按需填写额外字段（仅当标题非空时才会出现对应输入框）：

- **进行中**：可填 **背景**（这件事是什么）。
- **队列**：可填 **原因**、**阻塞**。
- **已完成**：可填 **结果**（交付/产出摘要）。

点 **添加** 后任务进入该列。若开启 **自动填写开始/结束日期**（页面上方勾选）：

- 新任务落在 **进行中** 时，可为空字段自动填 **开始日**；
- 新任务落在 **已完成** 时，可为空字段自动填 **结束日**。

---

## 4. 编辑已有任务

界面按列 **只把最常用的字段放在主区域**，其余收在 **「更多字段」** 折叠里；若折叠区内已有内容，展开时会 **自动打开**，避免看不见从其他列带过来的信息。

| 列 | 主区域（标题下） | 「更多字段」内 |
|----|------------------|----------------|
| **进行中** | 背景、开始日 | 原因、阻塞、结束日 |
| **队列** | 原因、阻塞 | 背景、开始日、结束日 |
| **已完成** | 背景、结果 | 开始日、结束日、原因、阻塞（从前列带入时可在此改） |
| **也许/将来** | 仅标题 | — |

另外还有：

- **子任务**：每行一条，左侧勾选表示已完成该子项；可点 **+ 子任务** 追加
- **详情**（多行）：自由备注，会存成多条普通 bullet

若任务已 **结晶**（见下文），界面会提示 **已结晶（已摄入）**。

---

## 5. 在列之间移动

卡片底部有一排 **列名按钮**，点哪个就把任务 **移到那一列**（不是「完成状态」菜单，只是换列）。

移动时若开启 **自动填写开始/结束日期**：

- 移入 **进行中** 且开始日为空 → 填当天；
- 移入 **已完成** 且结束日为空 → 填当天；
- 从 **已完成** 移出 → 可清除结束日（取决于当前逻辑与开关）。

删除单条任务用右上角 **✕**。

---

## 6. 「已完成」列整理（批量）

展开页面上的 **「已完成」列整理**：

- 用多选框选中若干任务；
- **归档所选**：追加写入 `kanban-archive.md`，并从主看板 Done 列移除；
- **删除所选**：仅从 `kanban.md` 删除，不写入归档文件。

归档规则与 MCP/CLI 行为见 [kanban-archive.md](kanban-archive.md)。

---

## 7. 已完成 → 证据（AI）

用于把 Done 里的工作 **草案式** 映射到 `evidence-pool.yaml` 与 `skill-tree.yaml`（与 CLI `nblane ingest-kanban` 同一条合并路径）。

**前置**：在 `.env` 配置 `LLM_API_KEY` 等，使 **AI 功能** 可用。

**推荐流程**：

1. 在折叠区 **多选** 若干「已完成」任务。
2. 按需勾选 **允许 AI 更新节点状态**（不勾选则只合并证据与引用，状态变更更保守）。
3. 点 **生成草案**，等待模型返回 JSON 草案。
4. 在 **选择要应用的草案条目** 中，分别勾选要采纳的 **证据行** 与 **节点更新**；每条会尽量展示 **原文摘录** 与 **理由**（模型需按提示输出）。
5. 查看 YAML **预览** 与 **差异** 说明，确认无误后优先点 **应用所选条目**（只写入勾选部分）；必要时也可用 **应用完整草案**。
6. 若希望把本次参与摄入的 Done 任务标为已处理，可勾选 **应用后将来源「已完成」任务标为已结晶**，再应用。

**说明**：未勾选某条 **证据行** 时，依赖它的 **节点引用** 会在子集合并时被丢弃或告警，避免写入悬空引用。具体警告以页面提示为准。

## 8. 与命令行对照

| 你在 Web 上做的事 | CLI（示例） |
|-------------------|-------------|
| 编辑看板 | 直接编辑 `kanban.md` 或使用本页 |
| 已完成 → 证据 | `nblane ingest-kanban <档案名>`（整包合并；Web 可细分条采纳） |
| 把看板带进上下文 | `nblane context <档案名>`（默认含看板；可用 `--no-kanban` 排除） |

## 9. 常见问题

**Q：改完没出现在磁盘？**  
A：点 **保存**，或触发会自动保存的操作（如移动列、添加任务等）。若怀疑不同步，用 **从文件重新加载** 对齐磁盘。

**Q：AI 生成的升级我不全信？**  
A：用 **按条采纳** + **应用所选条目**；关闭 **允许 AI 更新节点状态** 可只加证据、不改 tier。

**Q：Done 太长拖慢页面？**  
A：用 **归档所选** 把历史挪到 `kanban-archive.md`，或 **删除** 不再需要保留的条目。
